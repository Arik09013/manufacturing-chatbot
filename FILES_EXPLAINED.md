## 📁 `config/` — the knowledge & standards (data, not code)
| File | Why · How · Helps |
|---|---|
| `welding_params.yaml` | **Why:** the standards windows (current/voltage/speed…) per material × process × thickness. **How:** YAML tables read by the parameter advisor. **Helps:** gives **grounded, real numbers** for "best settings" answers. |
| `parameter_effects.yaml` | **Why:** what each parameter does (raise current → deeper penetration…). **How:** YAML read for the sensitivity narration. **Helps:** the "explain why these settings" part. |
| `welding_knowledge.yaml` | **Why:** curated welding know-how (porosity, cost, gas choice…). **How:** 25 topics matched by `knowledge.py`. **Helps:** answers defect/quality/cost questions **without hallucinating**. |
| `cause_action_map.yaml` | **Why:** map each fault → likely causes + recommended actions. **How:** YAML read by `root_cause.py` / `recommend.py`. **Helps:** root-cause diagnosis and the action advice. |

## 📁 `src/` — the source code (the engine)

### `src/api/` — orchestration & serving
| File | Why · How · Helps |
|---|---|
| `pipeline.py` | **Why:** the brain that runs everything in order. **How:** intent guard → router → (param / anomaly / knowledge) → builds SHAP/LIME/attention payload. **Helps:** turns a question into a full structured answer. |
| `main.py` | **Why:** a REST API alternative to the UI. **How:** FastAPI with `/chat`, `/pipeline/raw`, `/health`. **Helps:** lets other programs call the bot. |

### `src/chat/` — language layer
| File | Why · How · Helps |
|---|---|
| `intent.py` | **Why:** reject off-topic questions before wasting work. **How:** keyword/regex scope classifier. **Helps:** the guardrail ("I'm a welding assistant…"). |
| `prompts.py` | **Why:** tell the LLM exactly how to phrase answers. **How:** system prompts + builders that inject computed facts. **Helps:** clear answers with **no invented numbers**. |
| `synthesize.py` | **Why:** turn structured results into plain language. **How:** calls Groq / Claude / Ollama; deterministic fallback if none. **Helps:** the human-friendly final reply. |

### `src/data/` — data layer
| File | Why · How · Helps |
|---|---|
| `generate_synthetic.py` | **Why:** create the welding dataset. **How:** simulates normal welding + injects 5 fault types (seed-fixed). **Helps:** realistic data without a real factory. |
| `loaders.py` | **Why:** load the CSVs cleanly and typed. **How:** pandas read + dtype casting + optional filter by station. **Helps:** every other module gets clean data. |
| `schemas.py` | **Why:** validate the data is well-formed. **How:** Pydantic models (allowed stations, value ranges). **Helps:** catches bad/garbage data early. |

### `src/preprocess/` — clean each modality
| File | Why · How · Helps |
|---|---|
| `sensor.py` | **Why:** raw sensor data is noisy. **How:** impute gaps → denoise → normalize → window. **Helps:** stable features for the model. |
| `logs.py` | **Why:** turn event logs into numbers. **How:** dedup + count events per type (alarms, warnings…). **Helps:** log evidence the model can use. |
| `notes.py` | **Why:** turn free text into machine-readable form. **How:** sentence-transformer embeddings. **Helps:** the note modality enters the pipeline. |

### `src/fusion/` — combine the modalities
| File | Why · How · Helps |
|---|---|
| `align.py` | **Why:** the 3 modalities have different timestamps. **How:** aligns sensor/log/note to the same time windows. **Helps:** they can be compared/merged correctly. |
| `fuse.py` | **Why:** make one input table from three sources. **How:** merges aligned features → `fused.parquet`; `load_fused()` reads it. **Helps:** the single multimodal feature set. |

### `src/model/` — detection
| File | Why · How · Helps |
|---|---|
| `anomaly.py` | **Why:** detect faults from fused features. **How:** RandomForestClassifier + feature-matrix builder. **Helps:** the core anomaly detector (F1≈0.95). |
| `train_anomaly.py` | **Why:** produce the trained model. **How:** fits the RF on labelled data, saves `anomaly.joblib`. **Helps:** rebuild the model after data changes. |
| `bert_detector.py` | **Why:** the fine-tuned-LLM detector + text builder. **How:** serializes a window to text, classifies with DistilBERT. **Helps:** the optional second detector. |
| `finetune_distilbert.py` | **Why:** fine-tune DistilBERT + compare to RF. **How:** trains on serialized windows, evaluates on a held-out split. **Helps:** the **LLM-fine-tuning** thesis objective. |

### `src/explain/` — explainability (XAI)
| File | Why · How · Helps |
|---|---|
| `shap_explainer.py` | **Why:** explain *which sensor* drove a fault. **How:** SHAP TreeExplainer on the RandomForest. **Helps:** XAI method #1 (exact feature attribution). |
| `lime_explainer.py` | **Why:** an independent cross-check of SHAP. **How:** LIME local surrogate model on perturbed inputs. **Helps:** XAI method #2 (confidence via agreement). |
| `attention_explainer.py` | **Why:** explain the *text* understanding. **How:** DistilBERT self-attention heatmap over a note. **Helps:** XAI method #3 (which words mattered). |

### `src/reasoning/` — decision logic
| File | Why · How · Helps |
|---|---|
| `param_advisor.py` | **Why:** compute the best-efficiency weld settings. **How:** physics formulas + grid search inside the standards window. **Helps:** the parameter-optimization deliverable (your advisor's focus). |
| `knowledge.py` | **Why:** answer general welding questions. **How:** keyword-match the question to `welding_knowledge.yaml`. **Helps:** grounds the knowledge route. |
| `root_cause.py` | **Why:** turn a fault + SHAP into likely causes. **How:** ranks `cause_action_map.yaml` entries by SHAP/log support. **Helps:** the "why did it happen" answer. |
| `recommend.py` | **Why:** suggest the corrective action. **How:** looks up the action for the fault type. **Helps:** the "what to do" answer. |
| `confidence.py` | **Why:** tell the operator how sure the system is. **How:** 0.7 × anomaly prob + 0.3 × SHAP agreement. **Helps:** the confidence badge / trust. |
